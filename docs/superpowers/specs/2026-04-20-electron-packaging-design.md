# Shelf Electron Packaging + E2E Tests

**Date:** 2026-04-20
**Status:** Approved pending review

## Problem

Shelf is a web app that requires the user to run `pm start shelf` and open a browser tab. To feel like a proper photography tool it should launch from the dock with its own window, bundle everything needed to run, open on macOS and Windows, and have automated tests that catch regressions like the rotation bug that slipped through.

## Goals

1. **One-click launch.** Shelf.app in Applications, Shelf.exe on Windows. No terminal, no port manager, no browser.
2. **Keep the elf.** The pixel elf mascot (blue face, gold ears) is Shelf's visual identity. Use it for the dock/taskbar icon.
3. **Cross-platform.** Same codebase builds `.dmg` (macOS) and `.exe` (Windows NSIS installer).
4. **Safe by default.** Native binaries bundled correctly. Paths resolved correctly in packaged mode.
5. **Automated verification.** Playwright E2E tests catch silent regressions (rotation, marking, stage detection).

## Non-goals

- Code signing / notarization (needed for distribution to strangers; not needed for personal install)
- Auto-updater (cool, add later if distributing)
- Linux builds (easy to add via electron-builder, but not requested)
- Pro features (ratings, compare view, smart filters — separate spec)
- Custom protocol handler (`shelf://...`), system tray icon, MSI installer

## Architecture

### Process model

```
┌─────────────────────────────────────────┐
│ Electron Main Process                   │
│  - BrowserWindow (UI)                   │
│  - Native menu, dialog, notifications   │
│  - Spawns server child process          │
│  - IPC handler for pick-folder          │
└─────────────────────────────────────────┘
                │ spawn
                ▼
┌─────────────────────────────────────────┐
│ Node child process                      │
│  - Express server (server/index.js)     │
│  - PORT = free-port found by main       │
│  - sharp, exiftool, dnglab CLI          │
└─────────────────────────────────────────┘
                ▲
                │ http://localhost:<port>
                │
┌─────────────────────────────────────────┐
│ BrowserWindow (renderer)                │
│  - Loads http://localhost:<port>        │
│  - window.shelf preload bridge          │
│  - Dev: http://localhost:5173 (Vite)    │
└─────────────────────────────────────────┘
```

Child-process Express isolates server crashes from UI crashes and keeps the existing server code unchanged in structure.

### Dev vs packaged modes

| Mode | Server | Renderer loads | HMR |
|---|---|---|---|
| `npm run electron:dev` | `npm run dev` (existing, Vite + Express via concurrently) | `http://localhost:5173` (Vite) | Yes |
| Packaged (`.app` / `.exe`) | Spawned child: `node server/index.js` with `NODE_ENV=production` | `http://localhost:<freeport>` served by Express | No |

In packaged mode, Express serves the Vite-built static files from `dist/` in addition to the existing API. In dev, Vite serves the client and proxies `/api` to the server.

### Port selection

Hardcoded `3457` is a failure mode (two instances collide, stale process holds port). Main process picks a free port via `get-port` (simple, zero-dep option: iterate trying `.listen()` on 0 and read back the assigned port). Port is passed to the server via `PORT=<n>` env var, to the renderer via the URL it loads.

## Key design decisions (from sonnet review)

### asar unpacking

electron-builder's default `asar: true` bundles `node_modules` into a single archive. Native binaries inside asar can't execute. Required config:

```json
"asarUnpack": [
  "node_modules/sharp/**",
  "node_modules/@img/**",
  "node_modules/exiftool-vendored/**",
  "node_modules/exiftool-vendored.pl/**",
  "node_modules/exiftool-vendored.exe/**"
]
```

Without this, thumbnails and EXIF silently fail on first packaged launch. Verified during the first test build.

### Folder picker via IPC, not osascript

`server/lib/platform.js` currently spawns `osascript` from the Express server process to show the native folder picker. In a packaged Electron app this may fail (child processes without UI entitlements on macOS). Fix: move pick-folder logic to Electron main via IPC.

- `window.shelf.pickFolder()` in preload.js exposes the IPC call to the renderer
- `electron/ipc.js` handles `pick-folder` using `dialog.showOpenDialog({ properties: ['openDirectory'] })`
- `client/src/main.js` tries `window.shelf.pickFolder` first, falls back to the existing `/api/pick-folder` endpoint (for non-Electron dev/preview)

The server endpoint stays for dev mode — no breaking change.

### dnglab PATH resolution

The packaged Electron process doesn't inherit the user's shell `PATH`. `which dnglab` will fail even though dnglab is installed.

Replace `findDnglab()` in `server/routes/convert.js`:

```js
const DNGLAB_CANDIDATES = [
  '/opt/homebrew/bin/dnglab',   // macOS Apple Silicon
  '/usr/local/bin/dnglab',       // macOS Intel
  '/usr/bin/dnglab',             // Linux-style
  'C:\\ProgramData\\chocolatey\\bin\\dnglab.exe', // Windows via Chocolatey
];
async function findDnglab() {
  for (const p of DNGLAB_CANDIDATES) {
    if (fs.existsSync(p)) return p;
  }
  return null;
}
```

Do **not** bundle dnglab (10MB Rust binary, platform-specific builds, updates separately). Homebrew on macOS / Chocolatey on Windows is the right distribution path.

If dnglab not found, the "Convert to DNG" button is disabled with a tooltip explaining the install command, rather than silently failing with a 501.

### Window state

Save `{x, y, width, height}` to `~/.shelf/config.json` via the existing `windowBounds` config key (allowlisted in the previous UX redesign). On launch, validate bounds against `screen.getAllDisplays()` before restoring — if the saved bounds are off-screen (external display disconnected), clamp to the primary display.

### Native menu

| macOS | Windows |
|---|---|
| Shelf (About, Preferences, Quit) | _no app menu_ |
| File: New Shoot ⌘O, Open Recent ▶, Scan Card, Close Window ⌘W | File: New Shoot Ctrl+O, Open Recent ▶, Scan Card, Close |
| Edit: Undo ⌘Z, native Cut/Copy/Paste/Select All | Edit: Undo Ctrl+Z, native Cut/Copy/Paste/Select All |
| View: Reload ⌘R (dev), Toggle DevTools ⌥⌘I (dev), Actual Size/Zoom In/Out | View: Reload Ctrl+R (dev), Toggle DevTools Ctrl+Shift+I (dev), Zoom |
| Window: Minimize ⌘M | Window: Minimize |
| Help: Keyboard Shortcuts ⇧/ | Help: Keyboard Shortcuts Shift+/ |

`File > Quit` is omitted on macOS (it belongs in the app menu). Platform differences handled via `process.platform === 'darwin'` branches in `electron/menu.js`.

"Open Recent" is populated dynamically from the existing `recentShoots` config. Dynamic menu rebuild on each session start.

### Native notifications + progress bar

- **Notifications:** Electron's `Notification` API. Fires on convert/sort/promote complete **only when app is unfocused** (detected via `BrowserWindow.isFocused()`). When focused, the existing in-app toast is enough.
- **Progress bar:** `win.setProgressBar(fraction)` — renders under the dock icon on macOS, in the taskbar on Windows. Used during DNG conversion (fraction = converted/total). Clears on completion (`setProgressBar(-1)`). Portable, semantically correct.
- **Dock badge is out** — badges are for unread counts, not progress.

### Mascot / app icon

Preserve the pixel elf. `client/public/favicon.svg` (blue face, gold ears) is the source. A build script rasterizes it to `electron/icon/source.png` at 1024x1024 via sharp. electron-builder generates `.icns` (macOS) and `.ico` (Windows) from that source PNG automatically.

Pixel-art aesthetic: when scaling up, nearest-neighbor (no smoothing) to preserve the crisp blocky look. Sharp's `kernel: 'nearest'` option.

The existing in-app elf mascot (`client/src/elf.js`, 6-row pixel grid) is untouched — continues to peek out from the header as before.

### Delete dead launchers

`start.bat` and `start.command` at the repo root launch the app via a terminal. These become confusing once Electron is the distribution mechanism. Delete both.

## Playwright E2E tests

Use `@playwright/test` with Electron support (`_electron.launch()` API). Tests run against dev-mode Electron, not the packaged build (faster iteration, still exercises the real app).

Tests:

1. **`launch.spec.js`** — App launches, welcome screen visible, help button in bottom-right corner.
2. **`marking.spec.js`** — Load fixture `test/fixtures/unsorted`. Press P on an image, verify `.card.status-keep` class applied. Press F, verify favorite. Press X, verify reject. Press U, verify unmarked. Also verify `/api/mark` requests fire (intercept via CDP session or server-side log).
3. **`stages.spec.js`** — Load fixtures at three different paths. Verify stage pill text matches: `unsorted/` → CULL, `Keeps - 04-2026 - Test/` → HEROES, `Keeps - 04-2026 - Test/Favorites/` → FINAL.
4. **`rotation.spec.js`** — (Regression test.) Load fixture, select image, press Option+Right, verify `POST /api/rotate` fired, verify thumbnail src changes (bust param appears).
5. **`sort.spec.js`** — Load fixture, mark 2 keeps, open Sort modal, enter name, confirm. Verify bridge card appears with "Pick Heroes" button. Verify clicking it navigates.
6. **`lightroom.spec.js`** — Mock `execFile` in server. Load a Favorites fixture, click "Edit in Lightroom", verify `execFile` was called with `open -a "Adobe Lightroom CC"` (macOS) or the right Windows equivalent.

Fixtures: 3 tiny DNG files (~1MB each, generated from a phone photo rather than a CR3 to avoid Canon licensing). Store as `.gitignore`-excluded binaries or create programmatically in test setup. DNGs work with exiftool-vendored, sharp, and all routes.

Drag-drop is not tested (Playwright can't synthesize real OS drag events reliably for Electron).

## Out-of-scope concerns we accept

- **Windows file associations:** Shelf won't appear in the "Open with..." right-click menu in Explorer. Fix would require post-install registry entries; out of scope.
- **macOS Gatekeeper warning:** First launch shows "Shelf can't be opened because Apple cannot check it for malicious software." Right-click → Open to bypass. Only needed because we're not code-signing.
- **Auto-updater:** You'll manually rebuild to get updates. Fine for personal use.

## Success criteria

1. Running `npm run electron:dev` opens Shelf in its own window with hot reload
2. Running `npm run electron:build` produces `Shelf-<version>.dmg` on macOS and `Shelf-Setup-<version>.exe` on Windows
3. The DMG opens to show Shelf.app with the pixel elf icon, draggable to Applications
4. Installed Shelf.app launches, loads a folder, marks images, sorts, converts DNG (if dnglab installed), opens Lightroom — all of today's features work
5. `npm test` runs 6 Playwright E2E tests, all pass
6. No server console errors on normal use
7. Window bounds persist across launches; off-screen bounds clamp to primary display
8. Sorting a big folder with app in background triggers a native notification on completion

## Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| asar unpacking missed for a native dep | High | Explicit `asarUnpack` list in electron-builder config; test packaged build early |
| Electron Node version mismatch with sharp/exiftool-vendored | Medium | Use `electron-rebuild` postinstall; pin Electron version |
| Playwright Electron launch flaky in dev | Medium | Use `_electron.launch({ executablePath: electronBinary })`, wait for `app-ready` event before queries |
| First-launch Gatekeeper block confuses user | Low | Document "right-click → Open" in a BUILDING.md file |
| dnglab not in PATH on first install | Low | UI disables button with install instructions in tooltip |
